What is Method CRM MCP Server?

The Method CRM MCP Server is a production-ready implementation of the Model Context Protocol (MCP) that enables AI assistants like Claude to interact with your Method CRM data. Instead of manually navigating the CRM interface, you can use natural language commands to query customers, create records, manage files, and automate workflows.

Method CRM is a popular CRM platform especially favored by QuickBooks users for its deep integration and customization capabilities. By connecting it to AI through MCP, you unlock powerful automation possibilities.

üí° What You Can Do

  • Query customers, contacts, and opportunities using natural language
  • Create and update CRM records through conversation
  • Upload and manage files attached to records
  • Set up automated event routines
  • Manage API keys and user permissions
  • Generate reports and insights from your CRM data

Prerequisites

Before getting started, ensure you have the following:

üîß System Requirements

  • Python 3.10 or higher
  • pip package manager
  • Git (for cloning)
  • Claude Desktop app (optional)

üîë Method CRM Requirements

  • Active Method CRM account
  • API access enabled
  • API key with appropriate permissions
  • Admin access (for API key creation)

Getting Your Method CRM API Key

  1. Log into your Method CRM account at method.me
  2. Navigate to Customize ‚Üí API Keys
  3. Click Create New API Key
  4. Set permissions based on your needs (Read, Write, or Full Access)
  5. Copy and securely store your API key

‚ö†Ô∏è Security Note: Never share your API key or commit it to version control. Always use environment variables for credential management.

Installation & Setup

Step 1: Clone the Repository

git clone https://github.com/avisangle/method-crm-mcp.git
cd method-crm-mcp

Step 2: Create Virtual Environment (Recommended)

# Create virtual environment
python -m venv venv

# Activate on macOS/Linux
source venv/bin/activate

# Activate on Windows
.\venv\Scripts\activate

Step 3: Install Dependencies

pip install -r requirements.txt

Step 4: Configure Environment Variables

Create a .env file in the project root:

# .env file
METHOD_API_KEY=your_api_key_here
METHOD_BASE_URL=https://api.method.me  # Optional, uses default if not set

Or set environment variables directly:

# macOS/Linux
export METHOD_API_KEY="your_api_key_here"

# Windows PowerShell
$env:METHOD_API_KEY="your_api_key_here"

Step 5: Test the Installation

# Run the server in stdio mode
python -m method_crm_mcp

# Or run with HTTP transport
python -m method_crm_mcp --transport http --port 8080

‚úÖ Success! If you see "Method CRM MCP Server started" without errors, the installation is complete.

Claude Desktop Configuration

To use the Method CRM MCP Server with Claude Desktop, you need to add the server configuration to your Claude settings file.

Locate Your Config File

# macOS
~/Library/Application Support/Claude/claude_desktop_config.json

# Windows
%APPDATA%\Claude\claude_desktop_config.json

# Linux
~/.config/Claude/claude_desktop_config.json

Add MCP Server Configuration

Add the following to your claude_desktop_config.json:

{
  "mcpServers": {
    "method-crm": {
      "command": "python",
      "args": ["-m", "method_crm_mcp"],
      "cwd": "/path/to/method-crm-mcp",
      "env": {
        "METHOD_API_KEY": "your_api_key_here"
      }
    }
  }
}

üìù Configuration Options

  • command: Python executable (use full path if needed)
  • args: Arguments to run the MCP server module
  • cwd: Working directory where the server is installed
  • env: Environment variables including your API key

Using UV Package Manager (Alternative)

If you prefer using UV for Python package management:

{
  "mcpServers": {
    "method-crm": {
      "command": "uvx",
      "args": ["method-crm-mcp"],
      "env": {
        "METHOD_API_KEY": "your_api_key_here"
      }
    }
  }
}

Restart Claude Desktop

After saving the configuration, completely quit and restart Claude Desktop. The Method CRM tools should now appear in Claude's tool list when you start a new conversation.

Features & Available Tools

The Method CRM MCP Server provides 20 comprehensive tools organized into 5 categories. Each tool is designed for specific CRM operations with full type safety and validation.

üìä Table Operations (5 tools)

Full CRUD operations on any Method CRM table with OData query support.

  • method_tables_query - Query tables with filtering, sorting, and pagination
  • method_tables_get - Retrieve a specific record by ID
  • method_tables_create - Create new records in any table
  • method_tables_update - Update existing records
  • method_tables_delete - Delete records from tables

üìÅ File Management (6 tools)

Upload, download, and manage files attached to CRM records.

  • method_files_upload - Upload files with base64 encoding
  • method_files_list - List files with optional filtering
  • method_files_download - Download file content
  • method_files_get_url - Get direct download URLs
  • method_files_update_link - Update file-record associations
  • method_files_delete - Delete files from the system

‚ö° Event Automation (4 tools)

Create automated workflows triggered by CRM events.

  • method_events_create_routine - Create automated routines
  • method_events_list_routines - List all configured routines
  • method_events_get_routine - Get routine details
  • method_events_delete_routine - Delete routines

üîë API Key Management (4 tools)

Manage API keys for external integrations.

  • method_apikeys_create - Create new API keys with permissions
  • method_apikeys_list - List all API keys
  • method_apikeys_update - Modify key permissions
  • method_apikeys_delete - Revoke API keys

üë§ User Information (1 tool)

Access authenticated user and account information.

  • method_users_me - Get current user details and permissions

Advanced Configuration

HTTP Transport Mode

For remote deployments or multi-user scenarios, use HTTP transport:

python -m method_crm_mcp --transport http --port 8080 --host 0.0.0.0

Environment Variables

Variable Description Default
METHOD_API_KEY Your Method CRM API key Required
METHOD_BASE_URL API base URL https://api.method.me
LOG_LEVEL Logging verbosity INFO
RATE_LIMIT_REQUESTS Max requests per minute 60

Rate Limiting

The server includes automatic rate limiting with exponential backoff. If you hit Method CRM's API limits, requests are automatically retried with increasing delays.

Response Formats

Tools support both JSON and Markdown output formats. JSON is ideal for programmatic processing, while Markdown provides human-readable output for AI conversations.

Troubleshooting

‚ùå "API Key Invalid" Error

Cause: The API key is missing, malformed, or revoked.

Solution:

  • Verify the METHOD_API_KEY environment variable is set correctly
  • Check for extra spaces or quotes around the key
  • Confirm the key is still active in Method CRM settings

‚ùå "Permission Denied" Error

Cause: API key lacks required permissions.

Solution:

  • Review API key permissions in Method CRM settings
  • Ensure the key has Read/Write access for required tables
  • Create a new key with appropriate permissions if needed

‚ùå "Rate Limit Exceeded" Error

Cause: Too many API requests in a short period.

Solution:

  • Wait a few minutes and retry
  • The server automatically handles retries with backoff
  • Consider batching requests for bulk operations

‚ùå Claude Desktop Not Detecting Server

Cause: Configuration file issues.

Solution:

  • Verify JSON syntax in claude_desktop_config.json
  • Ensure the cwd path is absolute and correct
  • Completely quit and restart Claude Desktop (not just close window)
  • Check Claude's logs for error messages

‚ùå "Module Not Found" Error

Cause: Dependencies not installed or wrong Python environment.

Solution:

  • Activate the correct virtual environment
  • Run pip install -r requirements.txt again
  • Use full path to Python in Claude Desktop config

Best Practices

üîí Security

  • Never hardcode API keys in code
  • Use environment variables for all secrets
  • Create dedicated API keys with minimal permissions
  • Rotate API keys regularly

‚ö° Performance

  • Use OData filters to limit result sets
  • Implement pagination for large queries
  • Batch operations when possible
  • Cache frequently accessed data

üß™ Testing

  • Use a sandbox/test Method CRM account first
  • Test with read-only operations initially
  • Verify data integrity after write operations
  • Monitor API usage in Method CRM dashboard

üìù Prompting

  • Be specific about table names and fields
  • Include record IDs when updating/deleting
  • Ask Claude to confirm before destructive operations
  • Review results before proceeding with bulk changes
